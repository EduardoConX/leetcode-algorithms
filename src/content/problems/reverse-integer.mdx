# Problem #7 - Reverse Integer

## Problem Summary
Given a signed 32-bit integer `x`, return the integer obtained by reversing its digits.  
If reversing `x` causes it to go outside the signed 32-bit integer range `[-2³¹, 2³¹ - 1]`, return `0`.

Example:  
~~~text
Input: x = 123
Output: 321

Input: x = -123
Output: -321

Input: x = 120
Output: 21
~~~

---

## Approach 1: Mathematical Reversal

### Algorithm
1. Initialize `rev = 0`.  
2. While `x` is not zero:  
   - Extract the last digit: `pop = x % 10`.  
   - Remove the last digit: `x = x // 10`.  
   - Before pushing the digit, check for overflow:  
     - If `rev > INT_MAX // 10` or (`rev == INT_MAX // 10` and `pop > 7`), return 0.  
     - If `rev < INT_MIN // 10` or (`rev == INT_MIN // 10` and `pop < -8`), return 0.  
   - Append the digit: `rev = rev * 10 + pop`.  
3. Return `rev`.  

### Pseudocode
~~~text
function reverse(x):
    INT_MAX = 2^31 - 1
    INT_MIN = -2^31
    rev = 0

    while x != 0:
        pop = x % 10
        x = x // 10

        if rev > INT_MAX // 10 or (rev == INT_MAX // 10 and pop > 7):
            return 0
        if rev < INT_MIN // 10 or (rev == INT_MIN // 10 and pop < -8):
            return 0

        rev = rev * 10 + pop

    return rev
~~~

- Time Complexity: O(n) where n is the number of digits in `x`.
- Space Complexity: O(1).

## Approach 2: String Reversal (Simpler but Less Pure)

### Algorithm
1. Convert `x` to a string.  
2. Reverse the digits.  
3. Restore the sign if negative.  
4. Convert back to integer and check 32-bit bounds.  

### Pseudocode
~~~text
function reverse(x):
    sign = 1 if x >= 0 else -1
    reversedString = reverseString(abs(x))
    result = sign * toInteger(reversedString)

    if result < -2^31 or result > 2^31 - 1:
        return 0
    return result
~~~

### Complexity  

- Time Complexity: O(n) where n is the number of digits in `x`.
- Space Complexity: O(n) due to string conversion.

---

## Key Insight  
Both methods are valid, but the **mathematical approach** is preferred in interviews because it avoids string conversions and demonstrates understanding of integer manipulation and overflow handling.
